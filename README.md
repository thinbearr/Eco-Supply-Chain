# Eco Supply Chain Optimization System

## 1. Project Overview
The **Eco Supply Chain Optimizer** is an advanced dynamic logistics kernel architected to rapidly compute shortest paths. The central optimization goal is calculating the mathematically perfect equilibrium minimizing delivery cost, total operational time, and ambient carbon emissions. Through Carbon-aware logistics and event-driven dynamic routing, it averts gridlock natively in real-time, operating efficiently as a drop-in replacement for physical supply chain infrastructure tests.

## 2. Features
- **Event-Driven Routing Engine:** Executes ultra-fast parallel `Dijkstra` / Depth-First-Search vectors to rapidly adapt weights. 
- **Live Network Visualization:** Configured precisely using embedded Cytoscape.js canvases for zoomable, fully interactive graph generation.
- **Simulation Mode / Dynamic Multi-Factor Data:** Implements background background generators mapping out localized congestion states and continuous node temperature variances every 5 seconds.
- **System Analytics Dashboard:** Full enterprise SaaS Recharts matrix spanning Live Cost Breakdowns, Carbon Emissions payloads, Node Congestion spikes, and Histograms plotting route dominance over cycles.
- **Hardware Integration:** Native listener bridges the PySerial gap targeting Arduino physical inputs parsing environmental string drops locally.

## 3. Installation
**Backend (Python/Flask Daemon):**
```bash
cd backend
python -m venv venv
venv\Scripts\activate   # Wait for environment
pip install -r requirements.txt
```

**Frontend (Vite / React Dashboard):**
```bash
cd frontend
npm install
```

## 4. Running the System
Split your terminal to spin up concurrent instances. 

**Run Core Backend:**
```bash
cd backend
python app.py
```

**Run React Render Layer:**
```bash
cd frontend
npm run dev
```
Navigate your browser strictly to `localhost:5173`. 

## 5. Hardware Integration Requirements
The backend routing daemon intrinsically binds to `/dev/ttyUSB0` (Linux/Mac) or `COM3/COM4` (Windows) serial ports natively. An Arduino emitting comma-delineated syntax frames handles direct variable replacement overwriting the fallback random integer generated payload:
> e.g. `<Temperature_Reading>,<Congestion_Percentage>` natively ingested every readline tick.

## 6. Repository Architecture
